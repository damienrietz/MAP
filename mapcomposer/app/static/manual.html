How to start a new execution

Once this splash screen disappears the system Control Panel window will show up. At the Moment, December 2014, only CMRE Optimal Asset Allocator algorithm is available (look to the left side of the Control Panel window under “Algorithms”), in future other algorithms might appear in this section. In order to start a new Optimal Asset Allocator execution, do the followings:

  1. Click on the at end of the row “Optimal Asset Allocator” in the “Algorithms” section in the left side of the “Control Panel” tab or Right click on at the end of the green lines in the “Outcomes” section (right side) and select "Start from this run" to reuse parameters of a previous successful run
  2. Click on "Ok" button to on the dialog box “Create a new Process Run”, then the data entry form should appear
  3. Start entering or editing the execution input parameters in the data entry form
  4. Click on "Execute" button in the bottom part of the data entry form
  5. Go back to the control panel tab to monitor the process execution progress

How to visualize the results

In the “Outcomes” section of the “Control Panel” the list of executed run is presented. The color of the row reports the status of the execution: yellow in progress, green terminated with success, red terminated with errors.

The user can visualize the result of the execution clicking on the map icon on the green lines. This will open a new tab containing a GIS-like viewer with all the input and output layers of the Optimal Asset Allocator.

If the process failed, the button will show details about the error occurred.

How to share data

The Visibility column shows, in the “Outcomes” section can be one of the followings:

  • A lock icon when the execution results are private: only the creator (owner) can visualize them
  • A world icon if the execution results are public, which means that are visible to every authenticated user

Clicking on the pen icon the user can change the visibility of the results of a run, e.g. from private to public or vice versa.
